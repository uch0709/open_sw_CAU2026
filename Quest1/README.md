## 1. `top` 명령어

**설명**  
`top`은 실시간으로 시스템의 프로세스와 자원 사용량(CPU, 메모리, 스왑 등)을 모니터링하는 **인터랙티브 명령어**입니다. 화면을 지속적으로 갱신하며 현재 시스템 상태를 한눈에 확인할 수 있습니다.

**주요 특징**  
- 기본 3초마다 자동 갱신
- CPU, 메모리 사용량 기준으로 프로세스 정렬 가능
- 실행 중 키보드 단축키로 프로세스 제어 가능 (`k`: kill, `r`: renice 등)
- 시스템 전체 부하와 개별 프로세스 상태를 동시에 확인

**주요 옵션**  
| 옵션 | 설명 |
|------|------|
| `-d 5` | 갱신 주기를 5초로 설정 |
| `-n 10` | 10번 갱신 후 자동 종료 |
| `-u username` | 특정 사용자 프로세스만 표시 |
| `-p PID` | 특정 PID 프로세스만 표시 |
| `-o %CPU` | CPU 사용량 기준 정렬 |

## 2. `ps` 명령어

**설명**  
`ps` (**p**rocess **s**tatus)는 현재 실행 중인 프로세스의 **스냅샷**을 한 번 출력하고 종료되는 명령어입니다. `top`과 달리 실시간이 아닌 특정 시점의 프로세스 목록을 보여줍니다.

**주요 특징**  
- 다양한 형식(BSD, GNU)으로 출력 가능
- 프로세스 필터링과 정렬이 용이
- `grep`과 함께 사용하면 특정 프로세스 검색에 강력
- 프로세스 계층 구조(`--forest`) 확인 가능

**주요 옵션**  
| 옵션 | 설명 | 예시 |
|------|------|------|
| `aux` | 모든 프로세스 상세 정보 | `ps aux` |
| `-ef` | 전체 프로세스 (트리 형태) | `ps -ef` |
| `-u user` | 특정 사용자 프로세스 | `ps -u sihyung` |
| `--forest` | 프로세스 계층 구조 표시 | `ps -ef --forest` |
| `-o` | 출력 컬럼 지정 | `ps -eo pid,comm,%cpu,%mem` |

## 3. `jobs` 명령어

**설명**  
`jobs`는 **현재 셸 세션에서 백그라운드로 실행 중인 작업(Job) 목록**을 보여주는 명령어입니다. `&`로 실행한 백그라운드 작업이나 `Ctrl+Z`로 정지된 작업을 관리할 때 사용합니다.

**주요 특징**  
- 셸의 Job Control 기능 지원
- 작업 상태(Running / Stopped) 확인
- `fg`, `bg` 명령어와 연동하여 강력한 작업 관리 가능
- Job 번호(%1, %2)로 작업 제어

**주요 옵션**  
| 옵션 | 설명 |
|------|------|
| `-l` | PID와 함께 상세 정보 출력 |
| `-p` | PID만 출력 |
| `-r` | 실행 중인 작업만 표시 |
| `-s` | 중지된 작업만 표시 |

## 4. `kill` 명령어

**설명**  
`kill`은 **프로세스에 시그널(Signal)을 보내는** 명령어입니다. 주로 프로세스를 종료하거나 제어하는 데 사용되며, 다양한 시그널을 지원합니다.

**주요 특징**  
- PID 또는 Job 번호로 대상 지정
- 정상 종료와 강제 종료 구분 가능
- `pkill`, `killall`과 함께 이름 기반 종료 가능
- `kill -l`로 모든 시그널 목록 확인

**주요 옵션 및 시그널**  
| 시그널 | 번호 | 의미 | 사용 예 |
|--------|------|------|---------|
| SIGTERM | 15 | 정상 종료 (기본) | `kill PID` |
| SIGKILL | 9 | 강제 종료 | `kill -9 PID` |
| SIGHUP | 1 | 데몬 재시작 | `kill -HUP PID` |
| SIGSTOP | 19 | 일시 정지 | `kill -STOP PID` |
| SIGCONT | 18 | 재개 | `kill -CONT PID` |
